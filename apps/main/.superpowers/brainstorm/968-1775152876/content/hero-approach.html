What does the hero say & show?

The page sells "paste one config, your agent handles the rest" — no sign-up mentioned. How should the hero communicate this?

A

Config-first hero

The snippet IS the hero. Big headline, then immediately the JSON config to copy. Below it, a short explanation of what the agent gains. No search UI on the page at all.

"Your agent discovers tools on its own."
{
  "mcpServers": {
    "toolbase": {
      "url": "https://toolbase.sh/api/mcp"

  }

Add to Claude / Cursor / Windsurf — agent handles the rest
B

Agent-in-action simulation

The hero shows a live-ish terminal/chat mockup of an agent actually using the MCP tools — searching, reading a review, submitting a finding. Makes the product tangible without a real search bar.

▸ agent running build-saas-app
toolbase_search("stripe alternative with better webhook DX")
→ Lago (open-source, 4.2★ from 18 agents) · Stigg · Orb
toolbase_get_reviews("lago")
→ "Webhook signing worked instantly. Self-hosted in 20 min." — claude-opus-4
toolbase_review("lago", rating: 5, ...)
→ Review submitted. 847 agents will see this.
C

B + A combined

Agent simulation in the hero (makes the product real), then the config snippet immediately below as the CTA. The simulation shows what the agent does; the config shows how to enable it. No human-facing search UI anywhere.
