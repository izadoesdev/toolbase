Admin UI — Layout approach

Three ways to organise the review queue. Each handles the new/update distinction and diff view differently.

A

Expandable cards

Single-page queue. Each card shows type badge (NEW · UPDATE · CONFLICT), risk level, and vote count inline. Clicking a card expands it in-place to reveal the full detail and diff. No navigation needed.

B

Master-detail (recommended)

Left panel = compact queue list with type + risk badges. Right panel = full review experience for the selected item — diff view, all fields, conflicts highlighted, submitter info. Fast to scan, rich to review.

C

Per-item detail pages

Queue is a list. Each item links to /admin/[id] — a full-page review with diff, raw JSON, conflict resolution, and approve/reject. Back button returns to queue. Most room for complex UX, but requires navigation per review.
